<Lead>
It's easy to get started. Just use the following commands to set up and start your first project:
</Lead>

```bash
npm create jeasx@latest
cd jeasx-quickstart
npm run build
npm run dev
```

If you want to create a new project with a specific name, you can provide an additional argument:

```bash
npm create jeasx@latest my-awesome-project
```

If your project is successfully started, just visit `http://localhost:3000/` in a browser of your choice.

If you want to see the quickstart example in action, you can visit a deployed version of the quickstart template at [https://quickstart.jeasx.dev](https://quickstart.jeasx.dev). The contents of the quickstart template can be found at [GitHub](https://github.com/jeasx/jeasx-quickstart).

<Highlight title="Which JavaScript runtime should I use?">
Jeasx is developed and tested extensive with the **latest release** version of [Node.js](https://nodejs.org/) (currently Node 26.x). The **latest LTS** version (currently Node 24.x) is officially supported. Older versions are not supported, though they may work.

**Alternate JavaScript runtimes**

Jeasx is tested occasionally against [Bun](https://bun.sh) (`bun -b run [build|dev|start]`) and [Deno](https://deno.com) (`deno task [build|dev|start]`). At the moment no issues are known and all Jeasx examples are running fine on top of Bun ([bun.jeasx.dev](https://bun.jeasx.dev)) and Deno ([deno.jeasx.dev](https://deno.jeasx.dev)). If you encounter any issues, feel free to [report them](https://github.com/jeasx/jeasx/issues).

</Highlight>

## Install project with Docker support

If you want to use Docker, you can bootstrap a new project with additional resources (`Dockerfile`, `.dockerignore`) via the following flag:

```bash
npm create jeasx@latest -- --with-docker
```

## Install project with Vercel support

If you are going to deploy your project on Vercel, you can bootstrap a project with additional resources (`vercel.json`, `api/vercel.js`) via the following flag:

```bash
npm create jeasx@latest -- --with-vercel
```

For the initial setup of a Jeasx project on Vercel, you should select `Other` as the `Application Preset`.
