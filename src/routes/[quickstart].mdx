<Lead>
It's easy to get started. Just use the following commands to set up and start your first project:
</Lead>

```bash
npm create jeasx@latest
cd jeasx-quickstart
npm run build
npm run dev
```

If you want to create a new project with a specific name, you can provide an additional argument:

```bash
npm create jeasx@latest my-awesome-project
```

If your project is successfully started, just visit `http://localhost:3000/` in a browser of your choice.

If you want to see the quickstart example in action, you can visit a deployed version of the quickstart template at [https://quickstart.jeasx.dev](https://quickstart.jeasx.dev). The contents of the quickstart template can be found at [GitHub](https://github.com/jeasx/jeasx-quickstart).

<Highlight title="Which JavaScript runtime should I use?">
Jeasx is developed and tested extensive with the **latest release** version of Node.js (currently Node 25.x). The **latest LTS** version (currently Node 24.x) is officially supported and should work without issues. Older versions are not supported, though they may work.

**Support for alternate JavaScript runtimes**

- **Bun:** Jeasx is tested occasionally against latest [Bun](https://bun.sh) (`bun -b jeasx [build|dev|start]`) and all [Jeasx examples powered by Bun](https://bun.jeasx.dev) are working without any issues, so Bun can be considered as a stable platform for running Jeasx applications.

- **Deno:** Jeasx is tested occasionally against latest [Deno](https://deno.com) (`deno x jeasx [build|dev|start]`) and all [Jeasx examples powered by Deno](https://deno.jeasx.dev) are working without any issues, so Deno can be considered as a stable platform for running Jeasx applications. The warning `Not implemented: Server.setTimeout()` appears at startup. This [known issue](https://github.com/denoland/deno/issues/31221) is expected to be fixed in an upcoming Deno release.

If you encounter any issues, feel free to [report them](https://github.com/jeasx/jeasx/issues).

</Highlight>

## Install project with Docker support

If you want to use Docker, you can bootstrap a new project with additional resources (`Dockerfile`, `.dockerignore`) via the following flag:

```bash
npm create jeasx@latest -- --with-docker
```

## Install project with Vercel support

If you are going to deploy your project on Vercel, you can bootstrap a project with additional resources (`vercel.json`, `api/vercel.js`) via the following flag:

```bash
npm create jeasx@latest -- --with-vercel
```
